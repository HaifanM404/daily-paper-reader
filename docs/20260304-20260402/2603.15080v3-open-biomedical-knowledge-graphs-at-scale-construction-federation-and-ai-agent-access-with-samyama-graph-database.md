## 摘要
生物医学知识分散在孤立的数据库中——Reactome用于通路，STRING用于蛋白质相互作用，ClinicalTrials.gov用于研究注册，DrugBank用于药物词汇，DGIdb用于药物-基因相互作用，SIDER用于副作用。我们提出了三个开源生物医学知识图谱——通路知识图谱（来自5个来源的118,686个节点和834,785条边）、临床试验知识图谱（来自5个来源的7,774,446个节点和26,973,997条边）以及药物相互作用知识图谱（来自3个来源的32,726个节点和191,970条边）——它们构建于Samyama之上，这是一个用Rust编写的高性能图数据库。我们的贡献有三方面。首先，我们描述了一种可复现的ETL模式，用于从异构公共数据源构建大规模知识图谱，包括跨源去重、批量加载（Python Cypher和Rust原生加载器）以及可移植的快照导出。其次，我们展示了跨知识图谱联邦：将所有三个快照加载到单个图租户中可实现跨数据集的基于属性的连接。第三，我们引入了模式驱动的MCP服务器生成技术以支持LLM智能体访问，并在一个新的BiomedQA基准测试（40个药理学问题）上进行了评估：领域特定的MCP工具达到了98%的准确率，而模式感知文本到Cypher为85%，独立的GPT-4o为75%，且模式错误为零。所有数据源均为开放许可。在商用云硬件上，组合后的联邦图谱（790万个节点、2800万条边）加载时间约为3分钟；单知识图谱查询在80-100毫秒内完成；跨知识图谱联邦连接查询在1-4秒内完成。

## Abstract
Biomedical knowledge is fragmented across siloed databases -- Reactome for pathways, STRING for protein interactions, ClinicalTrials.gov for study registries, DrugBank for drug vocabularies, DGIdb for drug-gene interactions, SIDER for side effects. We present three open-source biomedical knowledge graphs -- Pathways KG (118,686 nodes, 834,785 edges from 5 sources), Clinical Trials KG (7,774,446 nodes, 26,973,997 edges from 5 sources), and Drug Interactions KG (32,726 nodes, 191,970 edges from 3 sources) -- built on Samyama, a high-performance graph database written in Rust.   Our contributions are threefold. First, we describe a reproducible ETL pattern for constructing large-scale KGs from heterogeneous public data sources, with cross-source deduplication, batch loading (Python Cypher and Rust native loaders), and portable snapshot export. Second, we demonstrate cross-KG federation: loading all three snapshots into a single graph tenant enables property-based joins across datasets. Third, we introduce schema-driven MCP server generation for LLM agent access, evaluated on a new BiomedQA benchmark (40 pharmacology questions): domain-specific MCP tools achieve 98% accuracy vs. 85% for schema-aware text-to-Cypher and 75% for standalone GPT-4o, with zero schema errors.   All data sources are open-license. The combined federated graph (7.9M nodes, 28M edges) loads in approximately 3 minutes on commodity cloud hardware, with single-KG queries completing in 80-100ms and cross-KG federation joins in 1-4s

*   **研究动机**：生物医学知识高度碎片化，分散在数十个独立的公共数据库中（如Reactome、STRING、ClinicalTrials.gov等），各自拥有不同的数据模式、标识符和访问方式。研究人员需要手动下载和整合这些数据，过程缓慢、易错且难以复现。
*   **整体含义**：本研究旨在解决这一“数据孤岛”问题，通过构建一套可扩展、可复现的开放生物医学知识图谱（KG）系统。该系统不仅整合了多源异构数据，还实现了跨图谱的联邦查询，并创新性地通过**模型上下文协议（MCP）** 为大型语言模型（LLM）智能体提供了高效、准确的领域知识访问接口。

#### **2. 方法论**
*   **核心思想**：
    1.  **模块化构建**：采用统一的ETL模式，从开放许可的数据源独立构建三个特定领域的KG（通路KG、临床试验KG、药物相互作用KG）。
    2.  **快照与联邦**：将每个KG导出为便携式快照文件（`.sgsnap`），可按需加载到同一个图数据库租户中。通过**基于属性的连接**（如共享的基因名 `gene_name`、药物名 `name`）实现跨图谱查询。
    3.  **AI代理优先访问**：基于图谱模式自动生成类型化的MCP工具，使LLM能够通过自然语言调用预定义的查询模板，而非直接生成Cypher查询语句。
*   **关键技术细节**：
    *   **ETL流程**：遵循“下载 → 解析与过滤 → 去重 → 批量加载 → 快照导出”的五阶段通用模式。使用共享注册表进行跨源实体去重。
    *   **联邦机制**：利用Samyama图数据库支持多快照加载的特性。查询时使用 `WHERE` 子句进行属性匹配来连接来自不同KG的节点（例如 `WHERE p.name = g.gene_name`）。
    *   **Schema驱动的MCP服务器生成**：
        *   服务器启动时自动从数据库读取模式。
        *   为每个节点标签和边类型生成基础工具（搜索、获取、计数）。
        *   根据YAML配置文件注册领域特定工具（如 `pathway_members`, `drug_side_effects`），这些工具本质上是参数化的Cypher查询模板。

#### ****8.** ****不足与局限******
1.**数据时效性与维护成本**:导出的快照是静态的时间点数据。对于快速更新的数据源(如ClinicalTrials.gov)，需要定期重新运行ETL流程以保持最新状态。
