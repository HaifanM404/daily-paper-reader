## 摘要
构建用于药物发现的最先进（SOTA）预测模型需要在工具、架构和训练策略上进行昂贵的搜索。目前基于大语言模型（LLM）的智能体可以通过大量的试错找到 SOTA 解决方案，但它们无法保留在此过程中积累的经验，因此在每个新任务上都要支付全部的搜索成本。我们提出了 DrugSAGE（自我演化智能体经验），这是一个跨任务积累并重用经验的框架，旨在高效构建 SOTA 药物发现模型。DrugSAGE 维护了一个跨任务存储库，包含经过验证的技能、关于有效策略的统计证据，以及反复出现的错误及其修复记录。在某些情况下，DrugSAGE 可以直接迁移有效的解决方案，而无需在测试时进行搜索。在 33 个分子性质预测任务中，DrugSAGE 在单任务设置下的九个 SOTA 智能体中排名第一。凭借从 16 个较小任务中积累的记忆，DrugSAGE 在跨任务评估设置下的 17 个预留任务中实现了 0.935 的平均归一化得分，并在零测试时搜索机制下优于所有基准智能体 10-30%。总之，我们的工作展示了跨任务记忆在高效开发药物发现 SOTA 模型方面的优势。

## Abstract
Building state-of-the-art (SOTA) predictive models for drug discovery requires expensive search over tools, architectures, and training strategies. Current LLM-based agents can find SOTA solutions through extensive trial and error, but they do not retain the experience accumulated along the way and therefore pay the full search cost on every new task. We propose \method (Self-evolving Agent Experience), a framework that accumulates and reuses experience across tasks to build SOTA drug discovery models efficiently. \method maintains a cross-task memory of verified skills, statistical evidence about effective strategies, and a record of recurring errors and their fixes. In some cases, \method transfers a working solution directly without test-time search. In 33 molecular property prediction tasks, \method ranks first among nine SOTA agents in a single-task setting. With memory accumulated from 16 smaller tasks, \method achieves an averaged normalized score of 0.935 on 17 held-out tasks in a cross-task evaluation setting and outperforms all baseline agents by 10-30\% in a zero-test-time search regime. In summary, our work shows the advantage of cross-task memory for efficient SOTA model development in drug discovery.

### 2. 方法论：核心思想与关键技术
DrugSAGE 的核心是**跨任务存储器（Cross-task Memory）**，它将经验整合进基于蒙特卡洛树搜索（MCTS）的循环中。
*   **核心思想**：通过维护一个持续进化的经验库 $Z = (R, H, Q)$，在搜索过程中引导智能体优先选择历史上表现良好的模型家族和优化策略。
*   **关键技术细节**：
    *   **技能库 (K)**：Explore Agent 通过搜索文献和 GitHub 自动构建可执行的技能库，无需人工预设起点。
    *   **解决方案存储器 (R)**：记录模型家族在不同任务上的表现，用于引导模型选择。
    *   **精炼存储器 (H)**：存储代码修改建议及其对性能的影响，帮助 LLM 提出更有效的优化方案。
    *   **执行存储器 (Q)**：记录环境配置、运行错误及修复方案，实现自动化的沙箱修复。
    *   **经验增强型 MCTS**：在选择模型家族时引入了带有“迁移项”的 UCB 公式，在保证理论收敛界限的同时利用历史经验。
    *   **DrugSAGE-ZERO**：一种“零测试搜索”模式，通过任务签名匹配，直接从存储器中检索并迁移最相似任务的验证方案。

### 3. 实验设计
*   **数据集**：
    *   **TDC-ADMET**：包含 22 个分子性质预测任务（吸收、分布、代谢、排泄、毒性）。
    *   **Polaris Hub**：包含 11 个额外的预留任务（ADME 和激酶抑制预测），用于验证跨任务迁移能力。
*   **对比方法**：
    *   **优化类智能体**：Autoresearch, ShinkaEvolve。
    *   **自动化 ML 智能体**：MLEvolve, AIRA-Dojo。

    *   **科学发现智能体**：Biomni, STELLA, Agentomics。
*   **实验场景**：
    1.  **单任务设置**：从零开始搜索，评估基础解决问题能力。
    2.  **跨任务设置**：利用 16 个小任务积累经验，在 17 个预留任务上评估效率提升。

### 4. 资源与算力
*   **硬件**：实验在单台 **NVIDIA L40S GPU** 上运行。
*   **时长**：每个智能体在每个任务上的墙钟时间限制为 **24 小时**。

*   **成本**：论文特别强调了 DrugSAGE-ZERO 的低成本，因为它在测试时几乎不产生 LLM 生成费用，仅需极少量的 Embedding 调用。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 33 个真实的药物发现任务，每个任务运行 5 个随机种子。
*   **消融实验**：进行了关于“Explore Agent 的作用”、“存储器组件贡献（Q vs 全量 Z）”以及“类比任务影响”的消融研究。
*   **充分性与公平性**：实验设计较为客观。作者通过归一化得分处理了不同量纲的指标，并确保所有基准智能体在相同的计算预算（B=20 次训练运行）下进行对比。此外，使用了预留数据集（Polaris）来验证经验的泛化性，避免了过拟合特定基准测试的风险。

### 6. 主要结论与发现
*   **性能领先**：在单任务设置下，DrugSAGE 的平均排名和获胜次数均位列第一。
*   **高效迁移**：DrugSAGE-ZERO（零搜索）在不进行任何新训练的情况下，性能优于所有进行了 20 次搜索迭代的基准智能体，性能提升达 10-30%。
*   **成本优势**：相比于其他智能体，DrugSAGE 在达到相同性能时所需的 LLM API 成本显著降低。
*   **理论保证**：证明了引入跨任务偏置项后的 UCB 策略依然保持了标准 UCB 的累积遗憾上界。

### 7. 优点
*   **经验闭环**：首次在药物发现智能体中实现了“探索-积累-复用”的完整闭环。
*   **自动化程度高**：Explore Agent 能够自动从原始文献和代码库中提取技能，极大地扩展了搜索空间。
*   **实用性强**：DrugSAGE-ZERO 模式为资源受限的科研场景提供了一种极具吸引力的“即插即用”方案。
*   **鲁棒的错误处理**：通过执行存储器 (Q) 实现了自动化的环境修复，减少了因琐碎代码错误导致的搜索中断。

*   **领域局限**：目前主要集中在分子性质预测（判别式模型），尚未扩展到分子生成或逆合成等更复杂的生成式任务。

*   **算力门槛**：虽然推理成本低，但构建初始经验库和运行 MCTS 搜索仍需要稳定的 GPU 资源支持。
